Robots.txt
DreamCatcher obeys robots.txt but does not currently respect HTML robot meta tags placed in the head of an HTML document. You can contain where DreamCatcher looks using the robots.txt web standard. DreamCatcher never follows links to hosts outside of the one you have specified. For more information about using robots.txt check out http://www.w3.org/TR/REC-html40/appendix/notes.html#h-B.4.1.1.
Tips for Error Handling Scripts
Whether you are using a custom script or the provided ErrorHandler.scpt script, what the script returns dictates how DreamCatcher responds. If the script returns true DreamCatcher assumes the error has been fixed so it removes the word from the misspellings list and reloads the page. However, if the script returns false DreamCatcher assumes the error has not been fixed so the word is not removed from the misspellings list and the page is not reloaded.
The provided script is commented and can give you insight into how it works. To open it simply double-click the file in the Finder. Additionally, Apple's Script Editor application has tools to help you debug errors that may occur when running your scripts.
DreamCatcher User Group
There is a DreamCatcher user group that you are welcome to join. There you can exchange error handling scripts and submit bug reports and feature requests. If you want to join the user group, go to the DreamCatcher Web site and follow the instructions for joining the DreamCatcher Users group.
